movable-view

movable-view​

Puede arrastrar y deslizar movable-view en la página. El componente movable-view debe estar dentro del componente movable-area y debe ser un nodo hijo directo. De lo contrario, el componente no puede moverse.

Notas:

  • movable-view debe establecer las propiedades de ancho y alto. De lo contrario, se utiliza el valor predeterminado de 10px.
  • Por defecto, movable-view utiliza posicionamiento absoluto, que no se puede cambiar. Los valores de las propiedades de arriba y izquierda son 0 px.
  • Cuando movable-view es más pequeño que movable-area, el rango de movimiento de movable-view está dentro de movable-area. Cuando movable-view es más grande que movable-area, el rango de movimiento de movable-view debe cubrir movable-area. Se consideran por separado la dirección del eje x y la dirección del eje y.

Código de ejemplo:

index.js

// API-DEMO page/component/movable-view.js
Page({
  data: {
    x: 0,
    y: 0,
  },
  onButtonTap() {
    const { x, y } = this.data;
    if (x === 30) {
      this.setData({
        x: x + 1,
        y: y + 1,
      });
    } else {
      this.setData({
        x: 30,
        y: 30
      });
    }
  },
});

Parámetros:

PropiedadTipoValor PredeterminadoDescripción
directionStringningunaLa dirección de movimiento de movable-view. Los valores válidos son "todos", "vertical", "horizontal" y "ninguno".
inertiaBooleanofalseEste campo especifica si movable-view tiene inercia.
out-of-boundsBooleanofalseEste campo especifica si movable-view puede moverse después de que el contenedor de vista esté fuera del área movible.
xNúmero0Este campo define el desplazamiento en la dirección del eje x, que se convierte en la propiedad left del componente. Si el valor de x no está dentro del rango movible, el componente se mueve automáticamente al rango movible.
yNúmero0Este campo define el desplazamiento en la dirección del eje Y, que se convierte en la propiedad top del componente. Si el valor de Y no está dentro del rango movible, se moverá automáticamente al rango movible.
dampingNúmero20El coeficiente de amortiguación, que se utiliza para controlar la animación que se activa cuando cambia el valor de x o y y la animación que se retira cuando el componente supera el rango. Un valor más alto conduce a un movimiento más rápido.
frictionNúmero2El coeficiente de fricción, que se utiliza para controlar la animación que se mueve debido a la inercia. Un valor más alto conduce a una mayor fricción e indica que el movimiento se detiene antes. Debe ser mayor que 0. De lo contrario, se utiliza el valor predeterminado.
disabledBooleanofalseEste campo especifica si se debe desactivar el componente.
scaleBooleanofalseEste campo especifica si se debe admitir el escalado con dos dedos. El área efectiva para gestos de escalado se encuentra dentro de movable-view de forma predeterminada.
scale-minNúmero0.5El valor mínimo del nivel de escala.
scale-maxNúmero10El valor máximo del nivel de escala.
scale-valueNúmero1El nivel de escala. Puede variar de 0.5 a 10.
animationBooleanofalseEste campo especifica si se deben usar animaciones.
onTouchStartEventHandle-El inicio del toque del dedo y este evento se pasa al nodo padre.
catchTouchStartEventHandle-El inicio del toque del dedo y este evento solo actúa sobre el componente y no se pasa al nodo padre.
onTouchMoveEventHandle-El dedo se mueve después del toque, el evento se pasa al nodo padre.
catchTouchMoveEventHandle-El dedo se mueve después del toque, el evento solo actúa sobre el componente y no se pasa al nodo padre.
onTouchEndEventHandle-La acción de tocar termina, el evento se pasa al nodo padre.
catchTouchEndEventHandle-La acción de tocar termina, el evento solo actúa sobre el componente y no se pasa al nodo padre.
onTouchCancelEventHandle-La acción de tocar se interrumpe, como recordatorio de llamada y ventanas emergentes.
onChangeEventHandle-El evento desencadenado durante el arrastre, evento. detail = {x: x, y: y, source: touch}, donde source muestra la razón del movimiento, por ejemplo, el valor es touch.
onChangeEndEventHandle-El evento desencadenado después del arrastre, event.detail = {x: x, y: y}.
onScaleEventHandle-El evento desencadenado durante el zoom, event.detail = {x, y, scale}.

onChange valor de retorno detail.source​
